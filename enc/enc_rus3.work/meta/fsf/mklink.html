MkLink

главная | FarStandardFunctions
Функция FSF.MkLink создаёт жёсткие и символические ссылки, связи каталогов, а также монтирует локальные диски на файловую систему. 
BOOL WINAPI MkLink(
  const wchar_t *Src,
  const wchar_t *Dest,
  enum LINK_TYPE Type,
  MKLINK_FLAGS Flags
);

Параметры

Src
Имя файлового объекта, на который создаётся ссылка.
Dest
Имя создаваемой ссылки.
Type
Тип операции. Одно из (перечисление LINK_TYPE):
ОперацияОписание
FLINK_HARDLINK создать жёсткую ссылку (hardlink)
FLINK_JUNCTION создать связь каталогов (junction)
FLINK_VOLMOUNT монтировать локальный диск на файловую систему
FLINK_SYMLINKFILE создать символическую ссылку-файл
FLINK_SYMLINKDIR создать символическую ссылку-каталог
FLINK_SYMLINK автоматически выбрать тип создаваемой символической ссылки
Flags
Флаг, одно из (тип MKLINK_FLAGS):
ФлагОписание
MLF_SHOWERRMSG выводить сообщения об ошибках
MLF_DONOTUPDATEPANEL после создания ссылки не обновлять панели
MLF_NONE нулевой флаг

Возвращаемое значение

1 - ссылка создана.

0 - ошибка при создании ссылки:

  • Для жёстких ссылок:
    • Src или Dest находятся на разных разделах;
    • раздел не NTFS;
    • раздел не локальный;
    • Src не существует или не является файлом.
    • Dest уже существует;
  • Для связей:
    • Src или Dest находятся не на локальных разделах;
    • раздел не NTFS 5.0;
    • Src не существует или не является каталогом;
    • Dest существует, но не пустой каталог.
  • Для монтирования дисков:
    • Src или Dest находятся не на локальных разделах;
    • раздел для Dest не NTFS 5.0;
    • Src не существует или диск является не локальным;
    • Dest существует, но не пустой каталог.

Примечание

  1. Ссылки создаются по следующим правилам:
    • жёсткие ссылки (hardlink) создаются только для файлов в пределах текущего NTFS-раздела;
    • связи (junction) создаются только для каталогов в пределах локальных NTFS-разделов;
    • монтирование локальных дисков на файловую систему только на NTFS-раздел.
  2. Если в качестве Src было передано значение, например L"C:", то вместо связи каталогов будет монтирован диск.
  3. Если для монтирования диска Dest оканчивается слешем, то в Dest будет создан каталог "disk_N" (где N - буква диска).
  4. В Windows 2000 напрямую делать связи на папки CD-ROM нельзя, но можно это ограничение обойти, смонтировав CD-ROM диск на папку NTFS-раздела (т.н. Mount Points). И уже из этой папки сделать нужную связь.
